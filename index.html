🔑 Key Engineering Decisions

1. Why Service Layer between Controller and Model?

Controllers should be thin — they only validate input and call services.
Business logic lives in the service layer so it can be tested independently and reused across routes.
This prevents the "fat controller" problem common in beginner Express apps.

2. Why PostgreSQL over MongoDB?

The data in this system is relational by nature. Using a document store would have meant either duplicating data or doing joins in application code — both worse than just using SQL properly.
PostgreSQL also gives us strong type guarantees and ACID compliance, which MongoDB doesn't offer by default.

3. Why Context API instead of Redux?

Redux adds real overhead — actions, reducers, thunks. For the scale of this app, Context + useReducer gives us the same predictable state management without the boilerplate. The decision was deliberate, not a shortcut.

4. JWT over Sessions

Sessions require server-side state, which breaks horizontal scaling. JWT keeps auth stateless — the server doesn't need to remember anything. Trade-off: tokens can't be invalidated early, so short expiry + refresh token pattern is used.

🛠️ Local Setup

# 1. Clone the repo
git clone <your-repo-link>
cd project-name

# 2. Setup backend
cd server
npm install
cp .env.example .env   # Fill in your values

# 3. Setup database
psql -U postgres -f database/schema.sql

# 4. Setup frontend
cd ../client
npm install

# 5. Run both
# Terminal 1
cd server \\\&\\\& npm run dev

# Terminal 2
cd client \\\&\\\& npm start

Environment Variables (server/.env):

PORT=5000
DATABASE\\\_URL=postgresql://localhost:5432/yourdb
JWT\\\_SECRET=your\\\_secret\\\_here
JWT\\\_EXPIRES\\\_IN=7d

📈 What I Actually Learned (Honest)

What I thought before What I learned
Backend = just write routes Controllers, services, models are separate concerns
SQL is slow Bad queries are slow. Proper indexes change everything
Auth is just login/signup Token lifecycle, expiry, and security matter a lot
Frontend state is simple Global state requires real architecture decisions
README is documentation README is communication — to humans, not machines
